2G, 3G, and 4G Data Log Analysis

Radio Frequency Engineer-Vocational Student Graduate Academy by KominfoMarquistech

a

Cellular Network Analysis Projects

The project is a part of the Radio-frequency Engineer training held by Kominfo. It happened for about 1 month and three times a week. The first task is should make a presentation about 2G network analysis and divided it into 6 groups. Each group contains 3-4 people. The method of grouping also goes the same with the 3G network except for 4G, because it should be finished individually as the mentor said. The log file is opened at network analytical software. After that, the signal quality of a certain location will be shown. From all of the coverage, we should identify which one spot had some issue. The issue means some points might have bad coverage, blank spot, high interferences, and so on. After defining the bad spot which is shown with some color parameters, it can be analyzed from the signal parameters. For 2G, it could be checked on Rx Lev and Rx Qual for basics. For further analysis can open the event menu and it will show the detailed event of a certain point. The big gap between all of the networks that I have mentioned above is the features that they provided, and for analytical can check on the parameters themselves. For 2G basic parameters, there are RxLev and RxQual. Then 3G, there will be RSCP and Ec/No. And for 4G, it is well known with RSRP, RSRQ, and SINR. The bad coverage or blank spot can cause some events like handover failure, dropped calls, call failure, or RRC connection failure. All those events might be caused by the signal quality between the serving cell and neighbor cells almost counting same so the UE will be confused to decide where it will be attached to a certain cell, the location has some obstacles like high building area, and other external factors. The external factor could be caused by high traffic, weather, and the distance from cells.
